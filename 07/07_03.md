# Exception Safety Analysis of Function `test`

```cpp
void test(const Person &person) {
    std::cout << "test : " << person.name() << '\n';

    if (person.grade() == 10 || person.salary() > 1'000'000) {
        save(Status::success, person.id());
    } else {
        save(Status::failure, person.id());
    }
}
```

Given that every member function of the `Person` class has the potential to throw exceptions, the following operations are exception points:

1. The invocation of `person.name()` — note that the preceding output `"test : "` will already have been sent to the stream if this throws
1. The call to `person.grade()`
1. The call to `person.salary()`
1. The call to `person.id()` (potentially in both branches)
1. The `save(Status::success, person.id())` function call
1. The `save(Status::failure, person.id())` function call
1. Any of the stream insertion operators (`operator<<`) might throw if:
   - `std::cout.exceptions()` has been configured to enable exceptions
   - Implementation-defined circumstances cause stream operations to fail

When execution proceeds without any of these throwing, the function completes its normal control flow path.
